# AIと協働してWebサイトを作る

(めんどくさいからAIに書かせた)

ellimia.net の改善プロジェクト（SEO、アクセシビリティ、パフォーマンス、PWA対応）を通じて得られた、AI（エージェント）を使い倒して高品質なプロダクトを作るためのコツのまとめ。特に、Google AntigravityとブラウザのAIを往復し、改善とコードレビューのサイクルを回すのがかなり良かった。

1. マスタープラン（箇条書きの羅列）を最初に渡す

AIに「いい感じにして」と丸投げするのではなく、人間側が「何を重視するか」の優先順位付きリストを提示するのが最も効果的。
今回の例：improvement plan.txt という形で、SEO、セキュリティ、UX、アクセシビリティの各項目を具体的にリストアップ。
これにより、AIが各タスクを細分化し、抜け漏れなく実行できる「レール」が敷かれる。
2. アクセシビリティとSEOを「AIの得意分野」にする

見栄え（デザイン）は人間の感覚が重要だが、セマンティックHTMLや構造化データ（JSON-LD）などはAIが得意とする「厳密な仕様」の世界。
aria-label の多言語対応や、モバイルメニューの Focus Trap（inert属性）の実装など、プロでも手間がかかる細かい仕様をAIに任せると、一気にサイトの品質（a11yスコア）が跳ね上がる。
「Search Consoleで警告が出ないように」「スクリーンリーダーで快適に読めるように」と制約を与えるのがコツ。
3. AIの「解釈のズレ」を軌道修正する対話

AIは時として、文脈を忘れたり、勝手にライトテーマにしたりする「迷走」をする。
履歴を振り返り、「ここは当初の計画と違う」「この機能は削除して」と明確に指摘することで、より洗練されたコードに進化する。
AIが出したバグを、人間が「このCSSプロパティで隠すべきでは？」とペアプログラミングの視点で指摘すると解決が早まる。
4. 使い捨てスクリプトによる自動化

全HTMLファイルの同じ箇所を書き換えるような「単純作業」は、AIにPythonスクリプトなどを書かせて一括処理させる。
例：全アセットのパスをルート相対パスに書き換える、全 aria-label を抽出して多言語化属性を付与するなど。
手作業による「置換ミス」を防ぎつつ、大規模な変更を数秒で完了できる。
5. AIクローラー対策とAI検索対応の共存

現代のWeb制作では robots.txt の書き方が重要。
「無断での学習（GPTBot等）」はブロックしつつ、「ユーザーの調べ物（ChatGPT-User等）」は許可するという、AIフレンドリーかつ権利を守る高度な設定も、AIに最新の仕様を反映させるとスムーズ。
6. 検証(Verification)とドキュメントの同期

コードを書いて終わりではなく、AIに「ブラウザで開いてスクリーンショットや録画を撮って証明して」と命じることで、デグレを防げる。
最後に README.md や Walkthrough.md を更新させることで、「今このサイトに何が実装されているか」を人間とAIの間で常に同期し続けることができる。
結論
AIは「高度な知識を持った、指示に忠実な作業員」。 人間が 「設計図（プラン）」と「品質の基準（a11y/SEO）」 をしっかりと握ることで、1人では数日かかる重厚な改善を、わずか数時間で、かつプロレベルの精度で完遂できるようになる。
